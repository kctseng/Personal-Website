Real Time Web

This project contains 5 real time data sources with Geo data. In the bottom section, three of the sources will be called through JSON, XML and JSONP format. They will be used to turn into points on map by using Google Maps API.



Instagram:

URL 1 (Click here)

Description: The link can return any data format (json, jsonp, xml) based on user requests. It will provide data feed to popular posts on Instagram. Location data are not always provided in Instagram posts, therefore the real time data has been filtered by if the post has location or not. The data will change rapidly in real time.


New York Times:

URL 2 (Click here)

Description: This data source provide recent newsfeed from New York Times US Section. It will update whenever there is a new article on NYT. The location data are provided under geo_facet in different format. Some articles will have empty geo_facet depending on the nature of the news. How detail the geo data are is also based on the news article. This data source is in xml format.


Foursquare:

URL 3 (Click here)

Description: the link provides venues search around Chicago on Foursquare site. It will be updated whenever there is new event added in Chicago to the FourSquare site. This data source is in json format.


Locr:

URL 4 (Click here)

Description: This link provides location for the popular pictures on Locr site. Geo info are under the latitude and longitude tag. This data source is in XML format.


TomTom:

URL 5 (Click here)

Description: This link provides locations for traffic information in each country. It will be updated whenever TomTom collected different traffic data. This data source is in json format.



Real Time Data Display: New York Times (XML)


This map will display the geo data receieved from New York Times. This is recent news in US posted on New York Times. Some places are not able to be found by Google depending on how they input position into the XML file. The center button next to each entry will center the map to that location. Clicking on the marker will have info window with article title, url to the article, and article's abstract.


In the New York Times data sourse, the locations were given in city name and/or state name. Instead of direct longitude and latitude, I had to geocode everything and make sure the center button will work to the right position after translate. The initial center point of the map is the center of the United States since the data source is most likely giving locations within the United States. This map will show where the news are happening all over the world and gives direct links to New York Times site after clicking on the markers on the map.
