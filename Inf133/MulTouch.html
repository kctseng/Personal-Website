Description

In this project, we modified the paint program. Originally, the program only works for regular drawing using one finger. We added a feature that will connect our touchpoints as vertices of a polygon and draw on the canvas when multitouch events occur. As shown in the video, we used a regular box and webcam with CCV and MT4J to complete this project. We had trouble in the beginning setting up the lighting and CCV correctly for it to produce the best quality blob, but after that, the rest project went relatively smoothly.

In our initial version of the code, after we added the multi touch feature, the paint program was not drawing when we only have single finger on it. Instead it only shows a dot where the touch is perceived, but never actually draws when the touch is moved on the canvas. This is the version shown in the video. We fixed this issue after we filmed the video, however, when we attempted to refilm the video, the webcam was no longer working properly, so we had to settle for what you see in the video. Another problem in the original code was that the image does not stay on the canvas when we started to produce another polygon. This was also fixed in our latest version of the code where the images will stay until we deliberately erase them with the eraser. This fix, however, is not shown in the video because of the camera problem we had, so we had to settle by turning in footage of the program that you see in the video. We still managed to create a multitouch program that adhere to all of the guidelines, however, we were not able to demonstrate our improvements because of the misbehaving camera. Future classes may wish to utilize a newer or slightly more sophisticated camera for the homemade touchpad.

For the process of coding, we first had to examine the drawing example in MT4J and understand how the program worked. The architecture basically includes the observer pattern we learned in class. The program listens for inputs from any input device. The input can be from the mouse, or it could come from another device like the gray-scale box that we built. Although inputs can come from different physical media, they are abstracted by the MT4J library as just abstract cursors. The program listens for cursor inputs, then according to the positions of the cursors, it draws a shape where the cursors are. Our extension to the drawing example connected all the cursors together by lines to make different kinds of shapes. In order to keep track of all the cursors that are currently giving inputs, and we keep track of a collection of cursor types; when one cursor changes positions, the changes will be reflected in the collection. We then have the collection of cursors and thus iterate through all the cursors in the collection, drawing lines to connect all the cursors and form polygons.
